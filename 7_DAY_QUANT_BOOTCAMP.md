# 7 天纯实战量化训练营

这不是一份“先背概念”的课程表，而是一份直接拿这个项目开练的 7 天实战计划。目标很简单：7 天后，你要能自己跑回测、改参数、做一次小研究、做一次因子筛选，并且在过程中顺手测试和优化项目代码。

整套训练只有 3 条规则：

1. 每天只解决一个明确问题，不要同时改很多东西。
2. 每次实验都要留下结果记录，不凭感觉下结论。
3. 每天结束前都跑同一组验证：`pytest -q`、`python -m compileall app`、`cd frontend && npm run build`。

## 开营前准备

先确保你能把项目跑起来：

```bash
python -m uvicorn app.main:app --reload --port 5001
cd frontend
npm run dev
```

浏览器里至少打开这 3 个页面：

1. `http://localhost:5001/docs`
2. 回测中心页面
3. 因子研究页面

你接下来 7 天会主要碰到这些文件：

- 后端入口：`app/main.py`
- 回测接口：`app/routers/backtest.py`
- 因子接口：`app/routers/factor.py`
- 回测执行：`app/services/backtest/run_service.py`
- 模拟盘编排：`app/services/backtest/paper_manager.py`
- 因子研究：`app/services/factors/service.py`
- 因子细节分析：`app/services/factors/detail_service.py`
- 因子组合分析：`app/services/factors/blend_service.py`
- 指标计算：`app/domain/market/technical_analysis.py`
- 回测前端接口：`frontend/src/modules/backtest/api.ts`
- 因子前端接口：`frontend/src/modules/factors/api.ts`
- 回测页面：`frontend/src/views/Backtest.vue`
- 因子页面：`frontend/src/views/tools/FactorResearch.vue`

## 每天通用动作

每天都按这个顺序做，不要跳：

1. 先写下今天只研究哪个问题。
2. 只改一个变量或一个代码点。
3. 跑实验，记录结果。
4. 看结果有没有真的变化。
5. 跑统一验证。
6. 写下今天的结论和下一步。

建议你从第一天开始就在根目录新建一个自己的实验记录文件，例如 `MY_TRAINING_LOG.md`。每天至少记 5 件事：

1. 今天改了什么
2. 改动前的基线结果
3. 改动后的结果
4. 是否通过验证
5. 你认为这个改动有没有价值

## 第 1 天：跑通全链路，完成第一次回测

### 今天目标

亲手做出第一条回测记录，并知道回测请求是怎么从页面走到后端、再回到页面的。

### 今天要做的事

1. 启动前后端。
2. 在回测页面做一次最简单的回测，不要追求高收益，只要能跑通。
3. 打开 Swagger，对照接口看这次页面调用了哪些接口。
4. 顺着读一条最短链路：
   - `frontend/src/modules/backtest/api.ts`
   - `app/routers/backtest.py`
   - `app/services/backtest/command_service.py`
   - `app/services/backtest/run_service.py`

### 今天必须产出

你要记录第一条基线回测结果，至少包括：

1. 策略名
2. 时间周期
3. 回测天数
4. 收益率
5. 最大回撤
6. 交易次数

### 今天结束前的验证

```bash
pytest -q
python -m compileall app
cd frontend && npm run build
```

### 今天学到的不是概念，而是一个事实

你会第一次真正知道：点一个“开始回测”按钮，代码里到底会发生什么。

---

## 第 2 天：做参数实验，不改代码也能开始量化研究

### 今天目标

学会做最基础的量化实验。你今天不改代码，只改参数，但要像研究员一样记录结果。

### 今天要做的事

固定策略不变，只做 4 组实验：

1. 固定其他参数，只改手续费
2. 固定其他参数，只改时间周期
3. 固定其他参数，只改回测天数
4. 固定其他参数，只改持仓比例

### 记录方法

给每一组实验做一张简单表格，列出：

1. 这次改了哪个参数
2. 参数从多少改到多少
3. 收益率变化
4. 最大回撤变化
5. 交易次数变化

### 今天必须产出

写出一句结论，例如：

“在当前策略下，手续费的影响比时间周期小，时间周期变化对交易次数影响最明显。”

### 今天结束前的验证

还是固定这 3 条：

```bash
pytest -q
python -m compileall app
cd frontend && npm run build
```

### 今天真正练到的能力

你开始学会量化里最重要的习惯：不凭印象下判断，而是做可重复实验。

---

## 第 3 天：第一次改代码，改一个指标或指标参数

### 今天目标

第一次真正改代码，并让结果发生可观察变化。

### 今天要做的事

打开 `app/domain/market/technical_analysis.py`，选择一个最熟悉的指标，建议从下面三个里选一个：

1. EMA
2. RSI
3. MACD

你的任务不是重写指标，而是做一个小改动，例如：

1. 改默认周期
2. 改计算窗口
3. 增加一个更明确的边界处理

### 实战要求

1. 改之前先记下基线回测结果
2. 改完后跑同一组回测
3. 比较结果是否真的变化
4. 如果结果没变化，找到原因，不要假装成功

### 今天必须产出

1. 一次真实代码改动
2. 一组改前改后的回测对比
3. 一句结论：这个改动到底有没有实际影响

### 今天结束前的验证

```bash
pytest -q
python -m compileall app
cd frontend && npm run build
```

### 今天真正练到的能力

你第一次把“代码改动”和“量化结果变化”连起来了。

---

## 第 4 天：做一个简单策略变体

### 今天目标

不再只调参数，而是做一个新的小策略版本。

### 今天要做的事

基于现有策略，做一个最简单的策略变体。不要发明复杂系统，选择一种最小改法：

1. 给原有信号加一个趋势过滤
2. 给原有策略加一个简单止盈或止损
3. 把单条件入场改成双条件入场

你会主要接触这些位置：

- `app/routers/backtest.py`
- `app/schemas/strategy_contract.py`
- `frontend/src/views/BacktestEditor.vue`

### 实战要求

1. 新策略要能在页面里发起回测
2. 旧策略和新策略都要跑一遍
3. 必须比较两个结果，不允许只跑新版本

### 今天必须产出

1. 一个新的策略版本
2. 新旧版本对比
3. 一句结论：新版本是改善了收益，还是只是减少了风险，还是根本没价值

### 今天结束前的验证

```bash
pytest -q
python -m compileall app
cd frontend && npm run build
```

### 今天真正练到的能力

你开始像一个真正做策略迭代的人，而不是只会点页面的人。

---

## 第 5 天：做第一次因子筛选

### 今天目标

开始用因子研究页做一次真正的“选特征”实验。

### 今天要做的事

1. 打开因子研究页面。
2. 用默认参数先跑一次完整分析。
3. 看排名靠前和靠后的因子分别是什么。
4. 从结果里挑出 3 到 5 个你认为比较稳的因子。
5. 观察组合结果和单因子结果有没有差异。

重点看这些文件：

- `frontend/src/views/tools/FactorResearch.vue`
- `frontend/src/modules/factors/api.ts`
- `app/routers/factor.py`
- `app/services/factors/detail_service.py`
- `app/services/factors/blend_service.py`

### 实战要求

今天不要自己发明新因子，只做筛选和对比：

1. 看相关性
2. 看稳定性
3. 看分层表现
4. 看组合后是否更平滑

### 今天必须产出

1. 一个你自己挑出来的因子小组合
2. 一段说明：你为什么留它们，为什么去掉另外一些

### 今天结束前的验证

```bash
pytest -q
python -m compileall app
cd frontend && npm run build
```

### 今天真正练到的能力

你开始接触“策略之前的研究”，这一步会让你慢慢具备研究感，而不是只会调回测。

---

## 第 6 天：把因子研究结果转成可执行回测

### 今天目标

把昨天筛出的因子组合，真的拿去做回测或模拟盘，而不是只停留在分析页面。

### 今天要做的事

通过因子研究页里的执行入口，做两件事：

1. 用一个因子组合发起因子回测
2. 如果你愿意，再发起一次模拟盘

重点看：

- `app/routers/factor.py`
- `app/services/factors/execution.py`
- `app/services/factors/paper_manager.py`
- `frontend/src/modules/factors/api.ts`

### 实战要求

你今天要比较的是两种东西：

1. 纯策略回测结果
2. 因子驱动回测结果

你不需要证明哪种一定更好，但你要回答这个问题：

“在当前项目里，因子研究结果能不能真的转成可执行交易逻辑？”

### 今天必须产出

1. 一次因子回测结果
2. 一段对比说明：它和你前几天做的普通策略回测，有什么本质不同

### 今天结束前的验证

```bash
pytest -q
python -m compileall app
cd frontend && npm run build
```

### 今天真正练到的能力

你开始把“研究”和“执行”连接起来，这才是量化项目真正有价值的地方。

---

## 第 7 天：做一次完整小闭环，并顺手优化项目

### 今天目标

独立完成一次从假设、实验、结果、改代码、再验证的闭环。

### 今天要做的事

今天你自己提一个小问题。问题必须足够小，能在一天做完，例如：

1. “给现有策略加趋势过滤后，能不能减少回撤？”
2. “在因子页面里，去掉高换手但不稳定的因子后，组合是否更稳？”
3. “某个页面结果展示太乱，我能不能顺手改得更容易比较？”

今天你需要完成完整动作：

1. 写下假设
2. 定位代码入口
3. 做一次最小改动
4. 跑实验
5. 看结果
6. 跑验证
7. 写结论

### 推荐今天顺手做的项目优化

从下面四类里选一个最小的优化点：

1. 让回测结果摘要更容易比较
2. 让因子排名更容易筛选
3. 让某个重复配置收成单一来源
4. 让实验记录更容易追踪

### 今天必须产出

你要写一份自己的“小型研究报告”，至少包括：

1. 我今天研究什么问题
2. 我改了哪里
3. 我怎么验证
4. 结果是什么
5. 这个改动值不值得保留

### 今天结束前的验证

```bash
pytest -q
python -m compileall app
cd frontend && npm run build
```

### 今天真正练到的能力

你已经不是在“学量化概念”，而是在用一个真实项目做研究、做实验、做工程闭环。

---

## 7 天结束后，你应该达到什么程度

如果你是认真做完这 7 天，而不是只看文档，你应该已经能做到：

1. 能独立完成一次回测
2. 能自己做参数实验
3. 能改一个指标或一个策略条件
4. 能看懂回测链路和因子链路
5. 能做一次最简单的因子筛选
6. 能把研究结果转成一次执行
7. 能在学习过程中顺手测试和优化项目

这时你还不是“资深量化”，但你已经脱离小白阶段了。更重要的是，你不会再把量化理解成一堆词，而会把它理解成一个可以动手、可以验证、可以迭代的过程。

## 训练营结束后的下一步

7 天之后，不要立刻去碰更复杂的模型。最好的下一步是继续做 3 轮小闭环，每一轮只解决一个问题：

1. 做一轮“策略参数稳定性”研究
2. 做一轮“因子筛选与组合”研究
3. 做一轮“页面与结果展示优化”改进

等你真的能连续做完这 3 轮，再去碰更高级的话题，例如参数搜索、滚动窗口验证、更多币种横向比较、策略模板扩展。
